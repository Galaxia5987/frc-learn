# Introduction to Programming

## What is Programming?

> "Computer programming or coding is the composition of sequences of instructions, called programs, that computers can follow to perform tasks. It involves designing and implementing algorithms, step-by-step specifications of procedures" - Wikipedia, [Computer Programming](https://en.wikipedia.org/wiki/Computer_programming)

At it's core, programming is the process of giving a computer a specific set of instructions to perform a task. Because computers cannot think(mostly) or understand human languages, we act as the logic makers, breaking down complex problems into logical commands that a machine can execute.

## How and what will we learn?
In FRC, We have four options when choosing our programming language:

* **C++**: This is a low-level, complex, language with a very steep learning curve. In FRC, it doesn't feature enough advantages over other languages, and it isn't the popular choice for teams.
* **Python**: This is a high level, simple, language with a very gentle learning curve. But, as a result of it's simplicity, it's harder to use when making larger and more complex projects like our robot code.
* **Java**: This is a high level, mostly simple, language with a noticeable learning curve. But, It has many useful features. Java also suffers from [boilerplate](https://en.wikipedia.org/wiki/Boilerplate_code) and it might be a bit tedious to type. It is also the most supported language for FRC, supporting a lot of libraries(extensions) that Python or C++ don't.
* **Kotlin**: Kotlin is a successor to Java, and has all of Java's features, but with simpler syntax(our grammer), and less [boilerplate](https://en.wikipedia.org/wiki/Boilerplate_code). Everything that can be written in Java, has it's equivalent in Kotlin.

**When writting our robot code, we will use Kotlin.**
As said, everything that can be written in Java can be written in Kotlin. But, sometimes, Kotlin simplifies the basic concepts that we need to know too much. To account for this, **we will first learn Java** thoroughly, and **then see the Kotlin equivalent**.
Additionally, As Kotlin and Java are interchangeable, when programming the robot you will see some Java code, and it is important that you will be able to read and understand it.

We will first understand and learn basic Java programs meant to run on a computer, and later on(After the learning Kotlin), we will see how we write code for our robot.

So, after making all of this clear, let's jump into learning Java!
