# OOP Patterns

OOP Patterns are standard solutions to common problems that occur when designing programs in Object-Oriented languages.
You can think of them like templates that help you solve programming issues in a smart way.

They don’t give you exact code, but they show you how to structure your code so that it’s easier to build, change, and maintain.

We have **a lot** design patterns, but in order not to bore you, we will only learn a few.

*P.S.* If you are interested in learning more about these patterns, the book "Design Patterns" is highly recommended, but can be a bit much, so be warned. You can find it [here](https://www.javier8a.com/itc/bd1/articulo.pdf) in pdf.
