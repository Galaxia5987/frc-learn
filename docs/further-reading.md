# Further Reading

If you finished reading everything here - Congratulations! This is definitely not a light guide, and we tackled some challenging concepts.
I hope you liked and learned from it, but it doesn't end here, coding(in FRC, and in general) takes a lot more than just knowing the fundamentals. It takes practice and experience. So make sure to get those as well.

## More Resources

If you liked this, and want to read more, here are some links to cool resources I personally like and recommend.

* [A Model-Based Approach to Shooting on the Move - Galaxia 5987](./files/shooting-on-move.pdf) - A paper we wrote last year about our approach to shooting on the move. I really like how it turned out.

* [Controls Engineering in FRC - Tyler Veness](https://controls-in-frc.link/) - A book all about control theory in FRC. Really cool book, however, the math might get a bit intense after the first few chapters.

* [Clean Code - Robert C. Martin](https://www.lkhibra.ma/books/clean-code.pdf) - A nice book about how to write clean readable code. It's quite long so maybe a skim would suffice for most.

* [Design Patterns](https://www.javier8a.com/itc/bd1/articulo.pdf) - A cool book about OOP design patterns. It's a bit on the long side aswell.
